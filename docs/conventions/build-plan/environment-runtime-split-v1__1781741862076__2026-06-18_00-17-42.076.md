# 6. WRITE PATHS

## Provider Context Pull

GET /api/environment/context/pull

Flow:

auth user
→ read current environment_snapshots row
→ read user_preferences.weather_mode
→ pull Open-Meteo provider context
→ normalize provider context
→ build environment_context_events insert row
→ filter persistence row
→ insert environment_context_events
→ build environment_snapshots update row
→ filter persistence row
→ update environment_snapshots

Behavior must remain equivalent.

---

## Provider Context Probe

GET /api/environment/context/probe

Flow:

auth user
→ read current environment_snapshots row
→ pull Open-Meteo provider context using shared adapter
→ return diagnostic payload
→ no DB write

Probe should not duplicate provider URL/fetch logic.

---

## OwnTracks HTTP Ingest

POST /api/ingest/location/owntracks

Flow:

HTTP request
→ read raw body
→ parse OwnTracks payload
→ read emitter device id header
→ lookup environment_emitters
→ insert environment_emitter_events
→ if valid location, upsert environment_snapshots

Behavior must remain equivalent.

Preserve HTTP-specific behavior:

- provider = "owntracks"
- source_mode = "owntracks"
- capture_mode maps trigger "u" to "manual"
- battery_pct is rounded integer
- pressure_kpa converts to pressure_hpa by multiplying by 10

---

## OwnTracks MQTT Ingest

MQTT message

Flow:

MQTT lifecycle worker
→ receive message
→ parse topic
→ parse OwnTracks payload
→ lookup environment_emitters
→ insert environment_emitter_events
→ if valid location, upsert environment_snapshots

Behavior must remain equivalent.

Preserve MQTT-specific behavior:

- provider = "owntracks"
- source_mode = "owntracks_mqtt"
- capture_mode preserves raw trigger or falls back to "mqtt"
- battery_pct preserves raw number
- pressure_kpa converts to pressure_hpa by multiplying by 10
- latest_emitter_raw includes topic and provider_user_id
